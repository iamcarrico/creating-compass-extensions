Compass has built a framework where you can easily extend the functionality of compass, and build your own mixins and functions to do anything!

Sass-y Code

Variables


$foo        : 'bar'   !default;

By adding a !default tag, the variable will only be set if it has not been set already

Use this within your extensions so variables can be set before or after the extension is called

Variables

  • Number - With or without units (1, 2.5, 100px)
  • String - With or without quotes (apples, "oranges", 'mangos')
  • Boolean - true / false
  • Null - null
  • List - Any combination of those with spaces or commas. (1em 2em 3em / alpha, beta, gamma)

Functions


@function addition($a, $b: 5) {
  @return $a + $b;
}

All function have at least one parameter, and have a @return to return a value.

You can set defaults to each parameter.

Mixins


@mixin right-to-left() {
  html[dir="rtl"] & {
    @content;
  }
}

Returns Sass code to be printed as CSS.

The @content represents all code within a curly bracket after the mixin is called.

Sass Provided Functions

Sass has numerous helper functions that can be found on their documentation. Examples:

  • unquote($var) / quote($var) Adds or removes quotes from a variable.
  • type-of($var) Will return the type of the variable (e.g. number)
  • unit($var); Returns the unit of a number (e.g. em)
  • join($list1, $list2); Merges several lists together
  • nth($list, $n) Will return the nth value in the $list.

Debugging

@warn will display an error on the command prompt, but will not stop the file from compiling. This is just to alert the user they are doing behavior that isn't advised.


@warn "This is an invalid response";

@debug displays similar to a warning, but just the debug message. Useful while coding.


@debug "This is an invalid response" $variable;

Building that Extension

Template to Begin

Team Sass has already built an extension for you to start with. Copy the repo and begin.

https://github.com/Team-Sass/Compass-Extension-Template

Gemspec file

The .gemspec file contains all the information rubygems.org and the system needs to ensure your gem gets installed properly.

The template has all the fields you need to fill in: the name of the gem, a description, etc.

Stylesheets

Compass will allow a Sass file to be pulled into a project when it is within the "stylesheets" directory.

Make sure the stylesheet is the same name as your extension, then you can do a normal include into your project

@import 'example';

Stylesheets

Within your stylesheets you can include your own mixins, functions, and other partials to perform any functionality you are looking for.

Extensions like breakpoint, singularity, and toolkit all are built on these stylesheets.

Templates

One of Compass's best features is to be able to create a template to start a theme from. The Aurora theme is a great example of this.

To create a template, within the "templates" folder create a folder named "project". This folder should contain all of your files you want within the template, as well as a "manifest.rb" file. The manifest file will allow you to say where each file should go to, as well as mark some to include variable names.

Templates

Example manifest.rb file


description "My Little Template"
file '../shared/template.php.erb', :to => "template.php", :erb => true

# Stylesheets
file 'style.scss',  :like => :stylesheet, :media => 'screen, projection', :to => 'style.scss'

# JavaScript
file '../shared/hammer.js', :like => :javascript, :to => 'hammer.js'

help %Q{
  You need help, contact the helpline!
}
welcome_message %Q{
  This is a welcome message!
}

Library

Within the lib folder is the ruby file that registers your gem with compass, and may contain some ruby code, if you need it.


require 'compass'

extension_path = File.expand_path(File.join(File.dirname(__FILE__), ".."))
Compass::Frameworks.register('example', :path => extension_path)

Testing and Deploying

Compiling and running locally

You can build and install your gem locally with just two commands:


gem build {your-gem-name}.gemspec
sudo gem install {your-gem-name}-{version-#}.gem

That's it! Your gem is installed locally, and you can test it!

Submit to Ruby Gems

Once you have an account on rubygems.org, you can publish your gem easily with just:


gem push {your-gem-name}-{version-#}.gem

Your gem is now pushed, and can be installed by anybody with:


sudo gem install {your-gem-name}

How / When to use Ruby

Sometimes... Sass can't do what you want it to do.

BUT RUBY CAN!

You can code your own extension to the objects that Sass has laid out. This is mainly useful for building functions that need Ruby-code to do what you need.

Ruby Examples

From Breakpoint:


module Sass::Script::Functions
  def is_breakpoint_list(breakpoint)
    result = false unless breakpoint.class == Sass::Script::List && breakpoint.separator.to_s == 'comma'
    Sass::Script::Bool.new(result)
  end
end
